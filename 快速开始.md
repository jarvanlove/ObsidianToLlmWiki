# 快速开始

这一页只回答 3 个问题：

1. 这套系统放在哪里
2. 第一次怎么开始
3. 日常怎么用

## 1. 这套系统放在哪里

你现在有两个仓库：

- `C:\Work\note\ObsidianToWiki`
  公开脚手架，放脚本、模板、规则、文档
- `C:\Work\note\ObsidianToWiki-private`
  私有知识库，放你的真实项目和个人知识

它们不是自动双向同步。

当前支持的是：

- 从公开脚手架同步一部分框架文件到私有库
- 私有内容不自动回写到公开仓库

## 2. 第一次怎么开始

第一次不要做太多，只做这两步。

### 步骤 A：把一个现成项目接进来

```powershell
python 00_system/scripts/attach_project.py --repo-root "C:\path\to\repo" --project "demo-saas"
```

执行后你会得到：

- 项目仓库里的 `wiki.context.json`
- 项目仓库里的 `AGENTS.md`
- 项目仓库里的 `CLAUDE.md`
- 私有知识库里的项目主干页

### 步骤 B：把一份真实资料收进来

```powershell
python 00_system/scripts/ingest_source.py --source "C:\path\to\prd.docx" --title "PRD v1" --project "demo-saas" --tags "prd,需求"
```

执行后你会得到：

- 原始文件副本
- 来源笔记
- 更新后的索引
- 更新后的日志

只要你真的跑过这两步，这套系统就已经能用了。

## 3. 日常怎么用

### 场景一：我要开一个项目

先跑 `attach_project.py`。

### 场景二：我有一份资料要收进去

先跑 `ingest_source.py`。

### 场景三：我想查之前怎么做过

先跑：

```powershell
python 00_system/scripts/search_wiki.py "你的关键词" --show-relations
```

### 场景四：我已经得到一个稳定结论，想记住

跑：

```powershell
python 00_system/scripts/file_back_query.py --title "你的标题" --question "你的问题" --conclusion "你的结论"
```

## 你暂时可以先忽略什么

先不用强记这些高级功能：

- 学习候选
- 候选升级
- 自动发现
- 来源推荐目标
- 关系同步细节
- 版本收口工具

这些是系统增强能力，不是第一次使用的前提。

## 你下一步应该做什么

现在最对的动作不是继续读，而是：

1. 选一个真实项目
2. 跑一次 `attach_project.py`
3. 找一份真实资料
4. 跑一次 `ingest_source.py`

然后再回来看 [[使用手册]]。
