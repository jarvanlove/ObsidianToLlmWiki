# 项目接入提示词

## 用途

当你在一个新的项目仓库里打开 Codex / Claude Code，希望它自动把项目工作接到中心 wiki 时，用这份提示词作为启动模板。

## 提示词模板

```text
你现在在一个项目工作区里工作，请自动把当前任务接入 ObsidianToWiki。

启动时先做这几件事：
1. 找到当前项目工作区中的 AGENTS.md 或 CLAUDE.md
2. 读取其中声明的 wiki_root、project_slug、project_index 等信息
3. 打开中心 wiki 的 README-zh.md、Home.md、index.md
4. 打开当前项目在 wiki 中对应的 索引.md、概览.md、架构.md、决策.md、任务.md、来源.md

执行任务时遵守这些规则：
1. 项目交付物留在当前项目仓库
2. 重要决策、结构变化、可复用经验要写回 wiki
3. 如果出现跨项目可复用知识，提升到 30_shared
4. 如果是问答分析或阶段总结，写入 40_outputs
5. 每次改动都补 backlinks
6. 追加 log.md
7. 重建索引

如果当前项目工作区还没有 AGENTS.md / CLAUDE.md，请先生成一个最小 bootstrap，再继续。
```

## 建议写入工作区的最小 bootstrap

```text
wiki_root: C:\\Work\\note\\ObsidianToWiki
project_slug: <project-slug>
project_index: 20_projects/active/<project-slug>/索引.md
project_overview: 20_projects/active/<project-slug>/概览.md
project_tasks: 20_projects/active/<project-slug>/任务.md
```
