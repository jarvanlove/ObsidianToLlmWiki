# 项目首次接入提示词

## 用途

当你在一个**还没有接入 wiki 的新项目仓库**里打开 Codex 或 Claude Code，需要先把项目桥接到你自己的私有 vault 时，用这份提示词。

它只负责**第一次接入**。接入完成后，后续任务改用现有的项目接入模板。

## 提示词模板

```text
你现在在一个新的项目仓库里工作，这个项目还没有接入私有 wiki。

请先完成“首次接入”：

1. 自动识别当前项目的仓库根目录和项目名
2. 在当前项目仓库根目录创建最小 `AGENTS.md` 和 `CLAUDE.md`
3. 在这两个文件里明确写入：
   - `wiki_root: <your-private-vault-root>`
   - `project_repo_root: <当前项目仓库根目录>`
   - `project_slug: <项目名>`
4. 在你的私有 vault 中为这个项目创建或更新项目页闭环：
   - `索引.md`
   - `概览.md`
   - `架构.md`
   - `决策.md`
   - `任务.md`
   - `来源.md`
5. 把当前任务的第一份工作内容登记到对应项目页，而不是只留在聊天窗口
6. 如果已经有适合的项目页，就在现有页面上继续更新，不要重复建新项目
7. 只有项目专属内容留在项目页，可复用经验再提升到 `30_shared`

首次接入完成后，请明确告诉我：
- 这个项目的 wiki 已经接通
- 接下来后续任务应该如何回写到项目页
- 哪些文件是项目工作文档，哪些文件是可复用知识
```

## 识别失败时的补充指令

如果 agent 无法自动识别项目仓库根目录，可以补这一句：

```text
请先用当前工作目录作为项目仓库根目录，再继续首次接入。
```
