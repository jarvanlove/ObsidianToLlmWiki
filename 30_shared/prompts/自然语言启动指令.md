# 自然语言启动指令

这个页面的目标是：让你以后尽量少记技术命令，只说需求。

## 项目接入

```text
帮我把当前项目接入 wiki。
如果当前目录已经是项目仓库，就直接完成项目接入，并让我后续在任意窗口都能继续使用项目 wiki。
```

## 项目开发中持续沉淀

```text
请在完成当前项目任务的同时维护项目 wiki。

要求：
- 先读取当前项目的 wiki.context.json 和 AGENTS.md / CLAUDE.md
- 完成任务
- 把有长期价值的结论写回项目 wiki
- 如果出现新资料，先摄入来源层
```

## 个人资料摄入

```text
把这份资料收进个人知识库。

要求：
- 保存原始文件
- 创建来源笔记
- 提炼关键观点
- 如果有长期价值，沉淀到个人页或共享页
```

## 项目资料摄入

```text
把这份资料收进当前项目。

要求：
- 保存原始文件到项目来源层
- 创建项目来源笔记
- 更新项目来源页
- 如有必要，更新概览、架构、决策或任务页
```

## 基于项目 wiki 回答

```text
基于当前项目 wiki 回答这个问题。

要求：
- 先读取项目索引、project.memory、任务页
- 只读取必要页面
- 回答时引用具体页面
- 如果结论稳定，写回项目 wiki
```

## 边开发边沉淀

```text
边开发边沉淀。

要求：
- 先读取当前项目的索引、project.memory、任务页
- 开发过程中如有新资料，先摄入项目来源层
- 开发结束后，把稳定结论写回概览、架构、决策、任务等项目主干页
- 如有可复用经验，提升到共享层
```

## 结论回写

```text
把这次结论记下来。

如果这是项目结论，就写回项目 wiki；
如果是通用经验，就提升到共享层；
如果只是阶段性分析，就写到 outputs。
```

## 使用原则

- 用户只说需求，不说脚本命令
- agent 自动判断这是项目接入、资料摄入、项目问答、持续沉淀还是结论回写
- agent 自动完成对应的页面更新、日志记录和索引维护
