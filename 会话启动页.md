# 会话启动页

这个页面的目标很简单：当你打开 Codex 或 Claude Code，不想重新组织语言时，直接复制下面的模板开始工作。

## 1. 摄入一份全局资料

```text
请按当前知识库规则摄入这份资料。

开始前先阅读：
- AGENTS.md（如果你是 Codex）或 CLAUDE.md（如果你是 Claude Code）
- README.md
- README-zh.md
- 使用手册.md

然后：
1. 必要时调用现有脚本，把原始资料放到正确位置，并创建来源笔记
2. 阅读资料内容，提炼核心观点、事实、约束、风险
3. 更新已有页面，而不是只新建孤立页面
4. 补充必要的双向链接
5. 如果产生了跨项目可复用知识，请提升到 30_shared 或 10_personal
6. 追加 log.md
7. 重建索引
```

## 2. 摄入到某个项目

```text
请把这份资料摄入 demo-saas 项目知识库，不要只保存来源文件。

要求：
1. 使用现有摄入脚本保存原始文件并创建来源笔记
2. 更新 demo-saas 项目的概览、架构、决策、来源、任务中真正受影响的页面
3. 如果产生了可复用知识，提升到 30_shared 或 10_personal
4. 补充必要的双向链接
5. 追加 log.md
6. 重建索引
```

## 3. 边做开发边维护项目 Wiki

```text
请在完成这个任务的同时，维护 demo-saas 的项目知识库。

要求：
1. 代码任务完成后，同步更新受影响的项目页面
2. 如果出现新的设计决策，补充到 决策.md 或新增 ADR 页面
3. 如果出现新的经验或踩坑，补充到 经验.md
4. 如果需要，新增项目笔记页并补充双向链接
5. 最后追加 log.md 并重建索引
```

## 4. 基于知识库回答并沉淀

```text
请基于当前知识库回答这个问题，并在有长期价值时把结论沉淀回 Wiki。

要求：
1. 先通过索引和搜索脚本定位相关页面
2. 只读取回答所需的最小页面集合
3. 输出回答时引用相关页面
4. 如果结论有长期价值，写入 40_outputs/analyses 或相关项目目录
5. 如有改动，追加 log.md 并重建索引
```

## 5. 做一次知识库体检

```text
请对当前知识库执行一次体检并直接修复能安全修复的问题。

要求：
1. 调用体检脚本
2. 检查孤儿页面、过期页面、缺少回链、应提升到共享层的项目经验
3. 对明确可修的问题直接修复
4. 生成体检报告
5. 追加 log.md
6. 重建索引
```

## 6. 提升跨项目经验

```text
请从 demo-saas 的项目知识库中识别可复用经验，并提升到 30_shared。

要求：
1. 阅读项目的概览、架构、决策、经验、来源
2. 识别哪些知识对其他项目也有价值
3. 创建或更新共享层页面
4. 从项目页加回链
5. 追加 log.md
6. 重建索引
```

## 使用建议

- 不要每次从零写需求，优先复制一条最接近的模板
- 先给 agent 模板，再给项目名、资料路径、你的关注点
- 让 agent 使用脚本，不要让它反复手工做同一种机械动作
