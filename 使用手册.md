# 使用手册

这份手册不追求讲全，只讲你日常会反复做的事情。

## 一句话理解

这套系统的工作方式是：

- 资料先进 `01_inbox` 或项目来源
- agent 帮你整理成项目页、共享页、输出页
- 以后靠搜索和索引再找回来

## 你只需要知道的目录

- `01_inbox/`
  新资料和临时材料
- `20_projects/`
  每个项目一套自己的知识区
- `30_shared/`
  跨项目共用的知识
- `40_outputs/`
  问答、分析、复盘结果
- `00_system/`
  脚本和模板

## 日常 4 个动作

## 1. 接入项目

适用场景：

- 你有一个现成代码仓库
- 希望 Codex / Claude Code 打开项目时能自动找到中心 wiki

命令：

```powershell
python 00_system/scripts/attach_project.py --repo-root "C:\path\to\repo" --project "demo-saas"
```

结果：

- 项目仓库会生成 `wiki.context.json`
- 项目仓库会生成 `AGENTS.md`、`CLAUDE.md`
- 私有 wiki 会生成项目主干页

## 2. 摄入资料

适用场景：

- 新需求文档
- 会议纪要
- 架构图说明
- 聊天记录整理稿

命令：

```powershell
python 00_system/scripts/ingest_source.py --source "C:\path\to\prd.docx" --title "PRD v1" --project "demo-saas" --tags "prd,需求"
```

结果：

- 原文件会保存
- 来源笔记会创建
- 日志和索引会更新

## 3. 搜索知识

适用场景：

- 我记得之前做过，但忘了在哪
- 我想知道某个项目依赖了什么
- 我想找权限、缓存、部署这类主题

命令：

```powershell
python 00_system/scripts/search_wiki.py "权限 设计" --show-relations
```

说明：

- 普通关键词搜索已经够大多数使用
- `--show-relations` 会额外带出项目关系和运行记忆

## 4. 写回结论

适用场景：

- 一次问答有长期价值
- 一次分析以后还会再用
- 一个项目结论不该只停留在聊天窗口

命令：

```powershell
python 00_system/scripts/file_back_query.py --title "权限方案评估" --question "当前项目该用 RBAC 还是 ABAC？" --conclusion "先用 RBAC，保留 ABAC 扩展点。" --project "demo-saas"
```

## 项目目录怎么理解

一个项目目录里，你只先记住这些页面：

- `索引.md`
  项目入口
- `概览.md`
  这个项目是干什么的
- `架构.md`
  系统结构和关键模块
- `决策.md`
  为什么这么做
- `任务.md`
  当前在推进什么
- `来源.md`
  资料入口
- `project.memory.md`
  给 agent 快速读的运行态摘要

其他像 `关系.md`、`风险.md`、`时间线.md`，是在你开始稳定使用后再逐步补强的。

## 如果你自己开一个新会话，应该怎么说

最简单的说法是：

```text
请按当前 wiki 规则工作。
先读 README-zh.md、快速开始.md、使用手册.md。
如果这是某个项目任务，再读取该项目仓库根目录的 wiki.context.json 和 AGENTS.md。
完成任务后，如果有长期价值，把结果写回 wiki。
```

更完整的模板见 [[会话启动页]]。

## 什么时候用私有库，什么时候用公开仓库

- 看脚手架、改规则、改模板、改脚本：在当前公开仓库
- 放真实项目内容、个人知识、原始资料、分析沉淀：在 `ObsidianToWiki-private`

## 你暂时不用管什么

先不用深究这些：

- schema 注册表
- 自动候选发现
- 候选升级
- 版本状态报告
- 私有库同步 manifest

这些是系统治理能力，不是你第一次用起来的门槛。
